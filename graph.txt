# IdeaConnect: Analytics and Graph Implementation Guide

This document outlines how IdeaConnect calculates user engagement metrics (votes, comments, views) and how these metrics are visualized using graphs within the application.

## 1. Calculating Metrics (Votes, Comments, Views)

To provide creators with accurate market validation data, the system tracks and aggregates user interactions.

### Implementation Steps

1.  **Database Schema (Supabase):**
    *   `ideas` table: Stores the core idea details.
    *   `votes` table: Records individual upvotes/downvotes, linking `user_id` to `idea_id` to prevent duplicate voting.
    *   `comments` table: Stores user feedback, linked to `idea_id`.
    *   `views` table (or a counter column): Tracks unique or total page views for an idea.

2.  **Data Fetching & Aggregation:**
    *   The application uses the `@supabase/supabase-js` client to query these tables.
    *   Instead of fetching all rows, it utilizes Supabase's built-in aggregation features (e.g., `{ count: 'exact' }`) to efficiently calculate totals.
    *   Example: `await supabase.from('votes').select('*', { count: 'exact', head: true }).eq('idea_id', currentIdeaId).eq('vote_type', 'upvote');`

3.  **State Management:**
    *   The fetched aggregates are stored in React state (e.g., `const [voteCount, setVoteCount] = useState(0)`) within components like `Dashboard.tsx` or `StatsGrid.tsx`.
    *   Real-time subscriptions (via Supabase Realtime) can be implemented to update these counts live as other users interact with the idea.

### What These Metrics Are Used For

*   **Market Validation:** High vote counts and active comment sections serve as quantitative proof of market demand.
*   **Ranking & Discovery:** The `BrowseIdeas` component uses these metrics to sort ideas by "Most Popular" or "Trending," increasing visibility for highly validated concepts.
*   **AI Context:** These metrics are fed into the Gemini AI engine (`aiValidation.ts`) to adjust the overall "Validation Score" based on real-world traction, not just theoretical analysis.
*   **Investor Signals:** Investors use these metrics to quickly identify ideas that have already achieved a degree of product-market fit.

## 2. Graph Implementation

To make the raw data easily digestible, IdeaConnect uses data visualization to show trends over time.

### Library Used
The application uses **Recharts** (`recharts: ^3.7.0`), a composable charting library built on React components and D3.js.

### Implementation Steps

1.  **Data Transformation:**
    *   Raw interaction data from Supabase (which includes timestamps) is grouped by day, week, or month.
    *   This data is transformed into a flat array of objects required by Recharts.
    *   *Example Data Structure:* 
        ```javascript
        const chartData = [
          { date: 'Mon', views: 120, votes: 15, comments: 3 },
          { date: 'Tue', views: 150, votes: 22, comments: 5 },
          // ...
        ];
        ```

2.  **Component Construction (`AnalyticsDashboardSection.tsx` / `StatisticsModal.tsx`):**
    *   **`<ResponsiveContainer>`:** Wraps the chart to ensure it scales correctly on both desktop and mobile devices.
    *   **`<LineChart>` or `<BarChart>`:** The core chart component, passing the `chartData` as its `data` prop.
    *   **Axes:** `<XAxis dataKey="date" />` and `<YAxis />` define the scales.
    *   **`<Tooltip>`:** Added to provide interactive hover states, showing exact numbers when a user hovers over a data point.
    *   **Data Series:** `<Line dataKey="views" stroke="#00BA9D" />` and `<Line dataKey="votes" stroke="#8B5CF6" />` map the specific metrics to visual lines on the graph.

3.  **Styling and Theming:**
    *   The charts are styled to match IdeaConnect's dark theme.
    *   Grid lines are made subtle (e.g., `stroke="#334155"`).
    *   Text colors for axes and tooltips are adjusted for high contrast against dark backgrounds.
    *   Brand colors (like the primary teal/green) are used for the most important data lines (e.g., Views or Validation Score).

### What The Graphs Are Used For

*   **Trend Analysis:** Creators can see if interest in their idea is growing, plateauing, or declining over time.
*   **Conversion Tracking:** By plotting "Views" alongside "Votes," creators can visually assess their conversion rate (how many people who see the idea actually like it).
*   **Reporting:** These visual graphs are key components of the exportable PDF reports generated via `jspdf`, providing professional summaries for stakeholders or potential investors.
